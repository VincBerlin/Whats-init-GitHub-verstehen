# Bilingual (DE/EN) Architecture — PHASE-6

Decision (OPEN-001): prepare the **architecture + a working language toggle** now; full
`/de` `/en` content routing is a later sprint. No broken canonical routes today.

## What's implemented
- `src/lib/i18n/index.ts`: `Locale` (`de` default, `en`), dictionary for chrome + hero + search.
- `LanguageToggle` (cookie `wii_lang`): switches locale, `router.refresh()` re-renders server chrome.
- Layout reads the cookie → sets `<html lang>` and translates nav; homepage hero/search use the dictionary.

## What's intentionally NOT done yet (future sprint)
- Per-locale routes `/de/...` `/en/...` and translated knowledge/legal content.
- `hreflang` + per-locale canonicals — **only add once real localized routes exist**, otherwise
  hreflang would point at non-existent pages (RISK-009). The cookie toggle does not create
  duplicate indexable URLs, so current SEO stays clean.

## Migration path to full i18n
1. Move content into per-locale data (`KnowledgeTopic.language` already in the data model).
2. Introduce `app/[locale]/...` segment or middleware locale routing.
3. Add `alternates.languages` (hreflang) + per-locale sitemaps.
4. Translate knowledge content with human review (no AI-slop, OUT-005).
